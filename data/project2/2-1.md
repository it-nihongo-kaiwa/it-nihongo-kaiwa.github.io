# 2-1 システムアーキテクチャ設計（Thiết kế kiến trúc hệ thống）

## 🗣️ 会話（Hội thoại）

**BrSE:** 本日はEコマースサイトの全体アーキテクチャ設計についてご説明いたします。  
*Hôm nay tôi xin giải thích về thiết kế kiến trúc tổng thể của trang E-commerce.*  

**KH:** 承知いたしました。まず、システム全体の構成図を見せてください。  
*Hiểu rồi. Trước tiên, hãy cho xem sơ đồ cấu trúc tổng thể của hệ thống.*  

**BrSE:** こちらが全体構成図です。フロントエンドはReact、バックエンドはNode.js、データベースはPostgreSQLを使用する、オーソドックスな3層構成になっています。  
*Đây là sơ đồ cấu trúc. Frontend React (Next.js), backend Node.js (Express), database PostgreSQL. Một cấu trúc 3 tầng (3-tier) chính thống cơ bản.*  

**KH:** フロントエンドとバックエンドの通信はどのように設計されていますか。  
*Giao tiếp giữa frontend và backend được thiết kế thế nào?*  

**BrSE:** シンプルかつ保守しやすいRESTful APIを採用します。JSON形式でデータの受け渡しを行います。  
*Sẽ áp dụng RESTful API vừa đơn giản vừa dễ bảo trì. Truyền và nhận data qua định dạng JSON.*  

**KH:** データベース設計について詳しく説明してください。  
*Xin giải thích chi tiết về thiết kế database.*  

**BrSE:** 主要なテーブルとして、users、products、orders、order_itemsなどを定義します。これらを正規化し、外部キーを用いてデータの整合性を保ちます。  
*Các table chính được định nghĩa gồm: users, products, orders, order_items. Chúng tôi sẽ chuẩn hóa (normalize) mớ này và dùng khóa ngoại (foreign key) để giữ tính toàn vẹn dư liệu.*  

**KH:** パフォーマンスを考慮した設計はどのようになっていますか。  
*Thiết kế có tính đến performance như thế nào?*  

**BrSE:** データベースには適切なインデックスを付与して検索速度を上げます。また、商品画像などの重い静的ファイルは、クラウドのストレージサービスに分離して配信します。  
*Sẽ đánh Index phù hợp lên DB để tăng tốc tìm kiếm. Ngoài ra, các file tĩnh nặng như ảnh sản phẩm sẽ được bóc tách ra lưu và phân phối bằng Storage service của Cloud (như AWS S3).*  

**KH:** セキュリティ面での設計はどうなっていますか。  
*Thiết kế về mặt bảo mật thế nào?*  

**BrSE:** ログイン後の中央認証・認可にはJWTを利用します。通信はすべてHTTPSとし、パスワードはBcryptで不可逆なハッシュにして保存します。  
*Để xác thực/phân quyền tập trung sau khi login sẽ xài JWT. Toàn bộ truyền tải mạng dùng HTTPS, password sẽ băm 1 chiều không thể đảo ngược bằng Bcrypt rồi mới lưu.*  

**KH:** マイクロサービス化の計画はありますか。  
*Có kế hoạch microservice hóa không?*  

**BrSE:** 今回の規模と予算では運用の複雑さが増すため、マイクロサービスは採用しません。代わりに、1つのコードベース内で役割を綺麗に分割した「モジュラーモノリス」の設計で開発し、将来の拡張に備えます。  
*Với quy mô và ngân sách lần này nếu dùng sẽ làm đội sự phức tạp khi vận hành, nên chúng tôi không áp dụng Microservices. Thay vào đó, sẽ dev theo hướng thiết kế Modular Monolith chia phân rõ vai trò trong cùng 1 codebase, nhằm dư phòng cho việc scale mở rộng sau này.*  

**KH:** 運用時の監視やデプロイメント戦略について説明してください。  
*Xin giải thích về lúc monitor vận hành và chiến lược Deployment.*  

**BrSE:** GitHub Actionsを用いて、コードをプッシュしたら自動でテストとデプロイが走るCI/CDパイプラインを構築します。監視についてはクラウドの標準機能を利用して、サーバーのCPU上限超過やエラー検知時にSlackへ通知を送ります。  
*Sẽ dùng GitHub Actions để dựng pipeline CI/CD sao cho cứ push code là auto chạy Test và Deploy. Về Monitoring sẽ xài tính năng chuẩn của Cloud (như CloudWatch) để bắn notify sang Slack nếu phát hiện Error hoặc CPU server bị quá rốn.*  

**KH:** 承知いたしました。無駄がなく理にかなったアーキテクチャだと思います。この設計で進めてよろしいでしょうか。  
*Tôi hiểu rồi. Một kiến trúc rất hợp lý không chút dư thừa. Có thể tiến hành theo thiết kế này luôn được chứ?*  

**BrSE:** はい、ありがとうございます。ただちに詳細なテーブル定義とAPI設計に着手し、来週までにご確認いただけるよう準備いたします。  
*Vâng, xin cảm ơn. Chúng tôi sẽ bắt tay ngay vào định nghĩa Table và thiết kế API chi tiết, chuẩn bị sẵn sàng để tuần sau anh/chị có thể confirm.*

## 📖 語彙（Từ vựng）

- 全体構成図（ぜんたいこうせいず）| sơ đồ cấu trúc tổng thể
- 3層構成（さんそうこうせい）| Cấu trúc 3 tầng (3-tier architecture)
- 保守しやすい（ほしゅしやすい）| dễ bảo trì
- 受け渡し（うけわたし）| truyền và nhận, giao nhận (data)
- 正規化（せいきか）| chuẩn hóa / Normalization (chia nhỏ bảng DB)
- 外部キー（がいぶキー）| Khóa ngoại (Foreign Key)
- 整合性（せいごうせい）| Tính toàn vẹn/Tính nhất quán (Consistency)
- インデックスを付与する（ふよする）| Gắn (đánh) chỉ mục Index
- 静的ファイル（せいてきファイル）| static files (file tĩnh như ảnh, css...)
- 中央認証・認可（ちゅうおうにんしょう・にんか）| xác thực và phân quyền tập trung
- 不可逆（ふかぎゃく）| không thể đảo ngược (một chiều)
- ハッシュにする | băm dữ liệu (hashing)
- 複雑さが増す（ふくざつさがます）| làm tăng thêm độ phức tạp
- モジュラーモノリス | Modular Monolith (Cấu trúc khối nguyên bản nhưng module hoá bên trong)
- コードベース | Code base (tập hợp mã nguồn dự án)
- デプロイメント戦略（せんりゃく）| Chiến lược deployment
- 自動で走る（じどうではしる）| Tự động chạy
- 上限超過（じょうげんちょうか）| vượt quá giới hạn/mức trần
- 無駄がなく（むだがなく）| không có gì thừa thãi
- 理にかなった（りにかなった）| hợp đạo lý, hợp lý, logic
