# Persuasive Rewrite Playbook (JA for IT)

## Goal
Turn understandable Japanese into convincing project communication:
- clear intent
- clear reason
- clear next action

## Rewrite pattern
1. State intent first.
2. State reason or business impact.
3. State concrete next step and owner.

## Useful connectors
- `まず`
- `そのため`
- `つきましては`
- `なお`

## Weak -> stronger examples
- `〜と思います` -> `〜と考えています`
- `できれば〜したいです` -> `可能であれば〜したく存じます`
- `問題ないです` -> `現時点では問題ございません`
- `確認してください` -> `ご確認いただけますでしょうか`
- `修正します` -> `本日中に修正し、再共有いたします`

## IT/business tone reminders
- Keep `です/ます` consistently.
- Prefer `要件定義`, `設計`, `実装`, `テスト`, `障害`, `リリース`, `レビュー`.
- Avoid mixing casual and formal forms in one dialogue turn.
